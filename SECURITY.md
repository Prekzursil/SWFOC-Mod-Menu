# Security Policy

## Supported Versions

This project tracks active development on `main`.

## Reporting a Vulnerability

Please do not open public issues for security-sensitive findings.

Report privately to the maintainer via GitHub security advisory or direct contact on GitHub.

When reporting, include:

- affected commit/version
- reproduction steps
- impact assessment
- suggested mitigation (if known)

We will acknowledge receipt and provide status updates as triage proceeds.

## Scope Notes

This repository is a trainer/editor for SWFOC single-player/mod workflows. Contributors should still follow secure coding practices for:

- file parsing and validation
- process and memory access boundaries
- update/manifest integrity
